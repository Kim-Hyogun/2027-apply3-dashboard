/grill-me

당신은 대학 입시 지원현황 자동화 시스템 및 실시간 공유 대시보드를 구축하는 수석 개발자입니다. 아래 요구사항과 아키텍처에 맞춰, 데이터 수집에서부터 보고서 엑셀/PNG 자동 생성, 깃허브 원클릭 동기화, 그리고 외부 공유용 보안 대시보드(Streamlit)까지 연동되는 전체 파이프라인을 구축해 주세요. 올해(2027학년도) 정리하는 정시 모집 지원기간은 2027년 1월4일(월)~1월20일(수)로 총 17일간이며, 작년(2026학년도) 정시 모집 지원기간은 2025년 12월29일(월)~2026년 1월14일(수)로 총 17일간입니다. 일단 구축 계획을 세우고 혹시 더 필요한 정보가 있다면 나에게 질문해서 정보를 더 보강하자.

---

### 1. 프로젝트 폴더 구조 및 데이터 명세
프로젝트 폴더 구조는 반드시 아래와 같이 생성 및 유지되어야 합니다.

- `[ROOT]/`
  - `input-this_years_support_status/` : 올해 N일차 일별 원본 엑셀 데이터 저장 (형식: `2027_apply3-Nday-input.xlsx`)
  - `input-last_years_application_status/` : 작년 최종 결과 보고서 저장 (`2026_apply3-report.xlsx` 고정)
  - `sample/` : 엑셀 서식 템플릿 보관 (`2027_report_1day.xlsx` 고정), 대시보드 화면구성 및 스타일 참조 (`dashboard_sample.png`)
  - `output-report/` : 자동 생성되는 결과 파일 경로 (`2027_report_Nday.xlsx`, `2027_report_Nday.png`)
  - `dashboard.py` : Streamlit 대시보드 어플리케이션
  - `report_generator.py` : 엑셀 보고서 생성 및 데이터 매핑 엔진
  - `excel_to_image.py` : pywin32를 이용한 고해상도 PDF 변환 및 이미지 크롭 모듈
  - `run_pipeline.py` : 통합 자동화 파이프라인 (N일차 자동 인식 및 가동)
  - `run_pipeline.bat` : 윈도우용 원클릭 자동 실행 파일 (백신 오탐지 우회)
  - `requirements.txt` : 라이브러리 의존성 명세 (openpyxl, pywin32, PyMuPDF(fitz), streamlit, pandas, plotly 등)

### 2. 컴포넌트별 상세 구현 사양

#### ① report_generator.py (엑셀 보고서 생성 엔진)
- **목적**: `sample/2027_report_1day.xlsx` 템플릿을 기반(입학정원 및 모집정원 고정)으로, 오늘 입력을 받아 누계/비교 수치가 반영된 `output-report/2027_report_Nday.xlsx` 파일을 생성합니다.
- **주요 기능**:
  1. **최신 일차 데이터 매핑**: `input-this_years_support_status/`에서 최신 `2027_apply3-Nday-input.xlsx` 파일을 읽어 정원내/외 데이터를 추출합니다.
  2. **전일 대비 증감**: N > 1 일 경우, 이전 일차 엑셀 보고서(`output-report/2027_report_(N-1)day.xlsx`)를 로드하여 정원내/외 전일 데이터를 추출하고 본문 T열, U열에 매핑합니다.
  3. **학과 명칭 변경 처리**: 작년 데이터와 매핑 시 예외 학과를 변환합니다. (예: 작년 `물리치료과` ➡️ 올해 `물리치료학과`)
  4. **날짜 및 시트명 동적 반영**: 1일차인 1월 4일(월)을 시작 기준으로 하여, N일차의 날짜를 계산해 본문 B3 셀 및 엑셀 시트명을 동적으로 변경합니다. (예: 2일차 ➡️ `2일차9월8일(화)17시`)
  5. **전년 당일 대비 및 최종 비교**:
     - `2026_apply3-report.xlsx` 파일에서 작년 N일차 및 최종 데이터를 가져와 V열, W열, J열, N열에 작성합니다.
     - 하단 42행(전년도 비교)과 44행(최종마감)에 작년 N일차 합계와 최종 합계를 기록합니다.
  6. **수식 꼬임 방지**: R열(계 전년최종)의 경우 템플릿 수식 누락을 방지하기 위해 각 학과 행(6~40행)에 대해 `=SUM(J{r}+N{r})` 수식을 코드로 명시하여 덮어씁니다.

#### ② excel_to_image.py (고해상도 이미지 크롭 모듈)
- **목적**: MS Excel 프로그램을 백그라운드로 실행시켜 인쇄 서식 영역을 100% 원본 그대로 보존한 PNG 이미지를 추출합니다.
- **주요 기능**:
  1. `pywin32` (`win32com.client`)를 사용하여 엑셀 어플리케이션을 백그라운드로 엽니다.
  2. 인쇄 범위를 `$B$2:$R$45`로 고정하고 세로 1페이지 크기에 딱 맞추도록 셋팅합니다. (`FitToPagesWide = 1`, `FitToPagesTall = 1`)
  3. 임시 PDF 파일로 익스포트하여 서식을 완전히 보존합니다.
  4. PDF 저장이 완료되면 `wb.Close(SaveChanges=True)`를 수행하여 엑셀 프로그램이 자체 연산한 수식 결과값(openpyxl은 연산 불가능)을 캐시 데이터로 파일에 강제 저장시킵니다.
  5. `PyMuPDF` (`fitz`) 라이브러리를 통해 임시 PDF의 첫 페이지를 300 DPI 고해상도 PNG 파일로 렌더링하고, 임시 PDF는 자동 삭제합니다.

#### ③ dashboard.py (Streamlit 대시보드)
- **목적**: 교직원 공유용 실시간 수시 지원현황 공유 웹 앱을 구현합니다.
- **디자인 가이드**: 다크 슬레이트 테마, 세련된 HSL 블루/골드/그린 컬러, 시인성이 뛰어난 메트릭 카드 사용.
- **핵심 요구사항**:
  1. **보안 로그인**: 최초 접속 시 비밀번호(`ipsi2027`)를 묻는 로그인 인터페이스를 탑재합니다. `st.secrets`가 설정되지 않았을 경우를 대비해 예외 처리 방어 코드를 심어 크래시가 나지 않도록 구현합니다.
  2. **동적 파일 감지**: 일차 선택 드롭다운은 캐시를 타지 않아야 합니다. `get_available_days()` 함수에 `@st.cache_data` 데코레이터를 적용하지 않아, 매일 새로운 3일차, 4일차 파일이 생성되면 대시보드 새로고침 시 드롭다운에 즉각 노출되도록 보장합니다.
  3. **요약 지표 (Metric Cards)**: 
     - 정원내 지원인원 및 경쟁률, 정원외 지원인원, 총 지원인원 합계, 작년 최종마감 대비 달성율(%)을 시각화합니다.
     - **합계 수치 보장**: 작년 최종 마감 인원 파싱 시, R41 셀의 수식 계산 꼬임에 대비하여 정원내 전년최종(J41)과 정원외 전년최종(N41)을 직접 더하는 방어 코드를 작성하여 정확히 **10,057명**으로 계산되도록 보장합니다.
  4. **트렌드 분석 및 그래프**:
     - 일차별 누적 지원자 추이 (올해 vs 작년 누적 라인 차트) 시각화.
     - 정원내 경쟁률 상위 TOP 10 학과 가로 바 차트 시각화.
     - 전년 대비 정원내 지원율이 하락한 학과만 분석하여 하락폭 TOP 10 및 상세 목록 표출.
  5. **학과별 상세 지원 현황표**:
     - 35개 학과의 세부 수치를 깔끔한 표로 보여주며, 테이블 최하단에 전체 누계를 합산한 **"계 (합계)"** 행을 추가해 시인성을 개선합니다.
     - 모바일이나 웹 뷰에서 불편한 세로 스크롤바가 생기지 않도록 데이터 길이에 비례해 높이(`height`)를 동적으로 자동 지정합니다.
  6. **보고서 이미지 뷰어 및 다운로드**:
     - 렌더링된 당일자 고화질 PNG 이미지를 화면에 띄우고, 엑셀 원본 파일(.xlsx) 및 이미지(.png) 다운로드 버튼을 제공합니다.

#### ④ run_pipeline.py / run_pipeline.bat (원클릭 자동 실행 파이프라인)
- **목적**: 사용자가 매일 엑셀 데이터 파일만 폴더에 넣고 더블클릭하면 깃허브 업로드까지 전체 자동화가 이루어지게 합니다.
- **워크플로우**:
  1. `input-this_years_support_status/` 내의 파일들을 검색하여 최신 일차(N일차)를 자동으로 감지합니다.
  2. `report_generator.py`를 가동하여 N일차 엑셀 보고서를 생성합니다.
  3. `excel_to_image.py`를 가동하여 N일차 PNG 이미지를 생성하고 엑셀 수식을 갱신합니다.
  4. 생성된 N일차 엑셀 및 PNG 파일을 Git stage에 추가(`git add`)하고, 커밋(`git commit -m "Auto-update: Day N Support Status Report"`)을 만든 후 원격지에 즉시 푸시(`git push origin main`)합니다.
  5. **배치 파일**: 사용자가 백신 프로그램 오탐지로 삭제될 걱정 없이 안전하게 더블클릭할 수 있도록, 동일한 가상환경의 python을 기동해 주는 `run_pipeline.bat` 파일을 루트에 생성합니다.

### 3. 최종 배포 및 외부 공유 설정 (Streamlit Cloud)
모든 개발이 완료되면 아래의 가이드라인에 맞춰 Streamlit Cloud 배포 설정을 지시해 주세요.
- **저장소 권한**: 소스코드 및 내부 데이터 보안을 위해 GitHub 리포지토리는 반드시 **Private(비공개)**로 유지해야 합니다.
- **외부 공유 권한 개방**: 비공개 저장소에서도 타 교직원들이 주소와 비밀번호(`ipsi2027`)만으로 접속하여 볼 수 있도록, Streamlit Cloud 어플리케이션 설정 `Sharing` 메뉴 ➡️ `Who can view this app`을 **`This app is public and searchable`**로 변경해 주어야 합니다.
- **Python 버전**: 패키지 의존성 충돌을 방지하기 위해 Streamlit Cloud 앱 생성 시 Python 버전을 반드시 **`3.12` 또는 `3.11`**로 명시하도록 안내해 주세요.
